# Project Docs

This folder is the documentation library for the reusable static-site framework. Legacy Defenders is the first active site profile implemented with it.

## Current Docs

- [System Overview](system-overview.md): project purpose, framework/site boundary, configuration model, and publishing model.
- [Architecture](architecture.md): directory map, build pipeline, templates, CSS, JavaScript, and validation layers.
- [Commands](commands.md): setup, `site.py`, scaffolding, Lighthouse, visual QA, and direct tool commands.
- [Site Profiles](site-profiles.md): active profile selection, profile files, scaffold defaults, and future-site workflow.
- [Content Guide](content-guide.md): Markdown pages, frontmatter, YAML data, compliance content, and content style.
- [Page Inventory](page-inventory.md): generated pages and the role each page plays in the site narrative.
- [Design System](design-system.md): CSS organization, tokens, components, page patterns, image rules, and QA rules.
- [Assets And Images](assets-and-images.md): original image workflow, optimized derivatives, hero images, and social assets.
- [Validation And QA](validation-and-qa.md): quality gates, visual QA, cognitive-load QA, and release checklist.
- [Deployment](deployment.md): GitHub Pages, GitHub Actions workflows, and publish flow.
- [Troubleshooting](troubleshooting.md): common build, CSS, JS, Lighthouse, visual QA, and deployment fixes.

## Archive
